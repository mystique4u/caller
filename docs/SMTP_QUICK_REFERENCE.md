# SMTP Server Quick Reference

## Connection Details

```
Host:     mail.yourdomain.com
Port:     587 (STARTTLS recommended)
          465 (SSL/TLS alternative)
          25  (Plain SMTP - not for apps)
Security: STARTTLS or SSL/TLS
Auth:     Required
Username: email@yourdomain.com
Password: (set during account creation)
```

## Quick Commands

### Create Email Account
```bash
# On server
/opt/services/mailserver/setup-email.sh add noreply@yourdomain.com

# Or from local machine
ssh root@SERVER_IP "/opt/services/mailserver/setup-email.sh add noreply@yourdomain.com"
```

### Generate DKIM Keys
```bash
# On server
/opt/services/mailserver/generate-dkim.sh

# Or from local machine
ssh root@SERVER_IP "/opt/services/mailserver/generate-dkim.sh"
```

### List Email Accounts
```bash
/opt/services/mailserver/setup-email.sh list
```

### View Logs
```bash
docker logs -f mailserver
```

## DNS Records (Add These)

### 1. A Record
```
Type:  A
Host:  mail.yourdomain.com
Value: YOUR_SERVER_IP
TTL:   3600
```

### 2. MX Record
```
Type:     MX
Host:     yourdomain.com
Priority: 10
Value:    mail.yourdomain.com
TTL:      3600
```

### 3. SPF Record
```
Type:  TXT
Host:  yourdomain.com
Value: v=spf1 mx ip4:YOUR_SERVER_IP ~all
TTL:   3600
```

### 4. DKIM Record
```
Type:  TXT
Host:  mail._domainkey.yourdomain.com
Value: (generated by generate-dkim.sh)
TTL:   3600
```

### 5. DMARC Record
```
Type:  TXT
Host:  _dmarc.yourdomain.com
Value: v=DMARC1; p=quarantine; rua=mailto:postmaster@yourdomain.com
TTL:   3600
```

## Testing

### Test Connection
```bash
telnet mail.yourdomain.com 587
```

### Check DNS Records
```bash
# Check A record
dig +short mail.yourdomain.com

# Check MX record
dig +short MX yourdomain.com

# Check SPF record
dig +short TXT yourdomain.com | grep spf

# Check DKIM record
dig +short TXT mail._domainkey.yourdomain.com
```

### Send Test Email
```bash
swaks --to test@mail-tester.com \
      --from noreply@yourdomain.com \
      --server mail.yourdomain.com:587 \
      --auth LOGIN \
      --auth-user noreply@yourdomain.com \
      --auth-password YOUR_PASSWORD \
      --tls
```

Then visit mail-tester.com to see your score.

## Application Integration

### Node.js (nodemailer)
```javascript
const nodemailer = require('nodemailer');

const transporter = nodemailer.createTransporter({
  host: 'mail.yourdomain.com',
  port: 587,
  secure: false,
  auth: {
    user: 'noreply@yourdomain.com',
    pass: 'your_password'
  }
});

await transporter.sendMail({
  from: 'noreply@yourdomain.com',
  to: 'user@example.com',
  subject: 'Test Email',
  text: 'Hello World'
});
```

### Python (smtplib)
```python
import smtplib
from email.mime.text import MIMEText

msg = MIMEText('Hello World')
msg['Subject'] = 'Test Email'
msg['From'] = 'noreply@yourdomain.com'
msg['To'] = 'user@example.com'

server = smtplib.SMTP('mail.yourdomain.com', 587)
server.starttls()
server.login('noreply@yourdomain.com', 'your_password')
server.send_message(msg)
server.quit()
```

### Environment Variables
```bash
SMTP_HOST=mail.yourdomain.com
SMTP_PORT=587
SMTP_USER=noreply@yourdomain.com
SMTP_PASS=your_password
SMTP_FROM=noreply@yourdomain.com
```

## Troubleshooting

### Email Not Sending
1. Check logs: `docker logs mailserver`
2. Verify DNS records: `dig TXT yourdomain.com`
3. Test connection: `telnet mail.yourdomain.com 587`
4. Check credentials

### Going to Spam
1. Verify all DNS records (SPF, DKIM, DMARC)
2. Test with mail-tester.com
3. Check PTR record (reverse DNS)
4. Warm up IP address gradually

### Connection Refused
1. Check if running: `docker ps | grep mailserver`
2. Restart: `docker restart mailserver`
3. Check firewall: Ports 25, 587, 465 must be open

## Firewall Ports

Ensure these ports are open:
- **Port 25** (TCP) - SMTP
- **Port 587** (TCP) - Submission (STARTTLS)
- **Port 465** (TCP) - SMTPS (SSL/TLS)

## Security Best Practices

1. ✅ Use strong passwords
2. ✅ Enable TLS/STARTTLS only
3. ✅ Monitor logs regularly
4. ✅ Limit accounts (only what you need)
5. ✅ Keep Fail2ban enabled
6. ✅ Regular backups

## Backup

```bash
# Backup mail data
tar -czf mailserver-backup.tar.gz \
  /opt/services/mailserver/config \
  /opt/services/mailserver/mail-data

# Restore
tar -xzf mailserver-backup.tar.gz -C /
docker restart mailserver
```

## Monitoring

```bash
# View real-time logs
docker logs -f mailserver

# Check mail queue
docker exec mailserver postqueue -p

# Flush queue (retry sending)
docker exec mailserver postqueue -f
```
